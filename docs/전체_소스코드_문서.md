# Lecture Master ERP · 전체 소스코드 문서

> 이 문서는 **기획자**와 **개발자** 모두가 읽을 수 있도록 작성되었습니다.  
> 화면이 어떻게 구성되어 있고, 메뉴가 어떻게 이어지며, 어떤 데이터를 쓰는지 한눈에 파악할 수 있습니다.

---

## 1. 전체 구조를 보는 방법 (모식도)

**전체 구조 모식도**는 브라우저에서 바로 볼 수 있는 HTML 파일로 제공됩니다.

- **파일 위치**: `docs/아키텍처_모식도.html`
- **보는 방법**: 해당 파일을 더블클릭하거나 브라우저로 열면, 화면 구성·메뉴 구조·데이터 경로가 도식으로 표시됩니다.

모식도에는 다음이 포함됩니다.

- 화면 전체 구성 (한 페이지에서 사이드바 + 메인 영역)
- 사이드바 메뉴 계층 (어떤 메뉴 아래에 어떤 하위 메뉴가 있는지)
- 상단 툴바·플러그인 버튼 구성
- 데이터가 어떤 JSON 파일에서 오는지
- 화면 전환 흐름 요약

## 2. 메뉴 구조도

### 2.1 사이드바 메뉴 (왼쪽 고정)

사이드바는 **한 단계만 펼쳐지는 메뉴**로 구성됩니다.  
클릭하면 해당 화면으로 전환됩니다.

| 메뉴 이름 | 종류 | 하위 메뉴 / 동작 |
|-----------|------|------------------|
| **B2B 강의리스트** | 펼치기/접기 가능 | **전체 강의**: 지금 보이는 목록 전체(일반 강의만). 기본 화면.<br>**제외된 강의**: 폐강이거나 제외 처리된 강의만 따로 표시. |
| **운영 대시보드** | 단일 메뉴 | 클릭 시 **운영 대시보드** 화면으로 전환. 테이블 대신 대시보드만 표시. |
| **내 강의리스트** | 펼치기/접기 가능 | **대시보드**: 내가 저장한 리스트들을 요약해서 보는 대시보드.<br>**[리스트 이름] (개수)**: 저장해 둔 리스트 하나를 선택하면, 해당 리스트에 들어 있는 강의만 테이블에 표시. |

**정리**

- **B2B 강의리스트** 아래: [전체 강의], [제외된 강의] 두 가지 보기.
- **내 강의리스트** 아래: [대시보드] + 저장된 리스트들이 나열됨. 리스트 이름 옆 숫자는 해당 리스트에 포함된 강의 개수.

---

### 2.2 메뉴가 이어지는 방식 (화면 전환)

- **B2B 강의리스트 → 전체 강의**  
  → 메인 강의 목록(폐강·제외 제외) 표시. 툴바에 “강의코드로 등록하기” 등이 보임.
- **B2B 강의리스트 → 제외된 강의**  
  → 폐강·제외 강의만 표시. 테이블 스타일만 “제외”용으로 바뀜.
- **운영 대시보드**  
  → 테이블 영역이 숨겨지고, 운영용 대시보드 화면만 표시.
- **내 강의리스트 → 대시보드**  
  → 테이블 영역이 숨겨지고, “내 강의리스트 대시보드”만 표시.
- **내 강의리스트 → [특정 리스트 이름]**  
  → 해당 리스트에 포함된 강의만 테이블에 표시. 툴바에 “리스트 관리”, “공유”, “업무요청”, “운영업무” 등이 나타남.

같은 페이지 안에서 영역만 바뀌는 방식이라, **주소는 바뀌지 않고** 화면 내용만 전환됩니다.

## 3. 버튼·기능 정리 (어떤 버튼 아래에 무엇이 있는지)

### 3.1 상단 툴바 (공통 또는 화면별)

| 위치 | 버튼/요소 | 설명 | 표시되는 화면 |
|------|------------|------|----------------|
| 왼쪽 | **검색창** | 강의명 또는 강의코드로 검색. 입력 후 검색 버튼 또는 Enter로 실행. | 테이블이 보이는 모든 화면 |
| 왼쪽 | **검색 초기화** | 검색어 지우고 원래 목록으로 복귀. | 검색어가 있을 때만 표시 |
| 왼쪽 | **필터 전체 초기화** | 컬럼별로 걸어둔 필터를 모두 해제. | **B2B 강의리스트 → 전체 강의**일 때만 |
| 오른쪽 | **페이지당 개수** | 10 / 20 / 50 / 100 / 200 / 500 / 1000개 중 선택. | 테이블이 보이는 모든 화면 |
| 오른쪽 | **저장** | 현재 선택한 강의들을 모아 “새 리스트”로 저장. 선택한 강의가 없으면 비활성. | 테이블이 보이는 모든 화면 |
| 오른쪽 | **플러그인 영역** | 아래 3.2처럼 메뉴별로 다른 드롭다운 버튼들이 표시됨. | 화면에 따라 다름 |

---

### 3.2 플러그인 영역 (드롭다운 버튼들)

플러그인 영역은 **지금 보고 있는 화면**에 따라 버튼 구성이 달라집니다.

#### (1) B2B 강의리스트 → 전체 강의일 때

| 버튼 이름 | 클릭 시 동작 |
|-----------|--------------|
| **강의코드로 등록하기** (드롭다운) | **강의코드로 등록하기**: 리스트 이름·강의코드·계약유형 등을 입력해 새 리스트로 등록.<br>**미정산 강의 등록하기**: 미정산 강의 목록에서 선택해 등록. |

#### (2) 내 강의리스트 → [특정 리스트] 선택했을 때 (공유 리스트 제외)

| 버튼 이름 | 클릭 시 동작 |
|-----------|--------------|
| **리스트 관리** (드롭다운) | **리스트 이름 수정**: 현재 리스트 이름 변경.<br>**강의 추가**: 다른 화면에서 고른 강의를 이 리스트에 추가.<br>**리스트 삭제**: 해당 리스트 삭제 (주의 표시). |
| **공유** (드롭다운) | **공유하기**: 팀/이름/아이디로 공유 대상 지정.<br>**공유된 사람 확인**: 이 리스트를 공유받은 사람 목록 확인. |
| **업무요청** (드롭다운) | **세금계산서 발행요청**: 세금계산서 발행 요청용 창(모달) 열기. |
| **운영업무** (드롭다운) | **세금계산서 발행**: 세금계산서 발행 관련 창(모달) 열기.<br>**강의료 정산파일 제작**: 정산 파일 제작 관련 창(모달) 열기. |

- **대시보드**, **제외된 강의**, **운영 대시보드** 화면에서는 플러그인 영역이 비어 있거나 표시되지 않습니다.
- **공유받은 리스트**를 보고 있을 때는 “리스트 관리”의 일부(예: 삭제)나 “업무요청/운영업무”가 없을 수 있습니다.

---

### 3.3 테이블 위에서 하는 동작

- **체크박스**: 강의를 선택/해제. 선택한 개수는 상단에 “선택: N개”로 표시되고, “저장” 버튼과 연동됩니다.
- **셀 선택**: 셀 클릭·드래그로 여러 셀 선택 (복사 등에 활용).
- **행 펼치기**: 일부 행은 “펼치기” 시 해당 강의의 상세 정보(연결된 Re챔프·챔프·통합 LCMS 등)가 아래에 펼쳐집니다.
- **비교 버튼**: B2B 강의와 Re챔프/챔프/통합 LCMS 정보를 한 번에 비교하는 창(모달)을 엽니다.
- **페이지 이동**: 테이블 하단의 “이전 / 1,2,3… / 다음”으로 페이지 이동.

## 4. 기능이 어떻게 동작하는지 (쉽게 정리)

아래는 “개발 용어 없이” 동작만 설명한 것입니다.

### 4.1 앱이 켜질 때

1. 여러 JSON 파일에서 강의 목록, 사용자 목록, 교재 목록, Re챔프·정산·챔프·통합 LCMS 정보 등을 한꺼번에 불러옵니다.
2. 메인 강의 목록은 “일반 표시용”과 “제외된 강의용”으로 나뉘어 저장됩니다.
3. 처음에는 **B2B 강의리스트 → 전체 강의** 화면이 보이고, 그 목록이 테이블에 표시됩니다.

### 4.2 검색

- 사용자가 **강의명** 또는 **강의코드**를 입력하고 검색하면,  
  **지금 화면에 기준이 되는 목록**(전체 강의, 제외된 강의, 또는 선택한 리스트) 안에서만 검색합니다.
- 검색 결과만 테이블에 표시되고, 페이지는 1페이지로 돌아갑니다.
- 검색 초기화 버튼으로 검색어를 지우면 다시 원래 목록이 보입니다.

### 4.3 필터

- 테이블 각 컬럼 제목 옆의 **필터 아이콘**을 누르면, 그 컬럼에 있는 값 목록이 나타납니다.
- 원하는 값만 골라서 “이 컬럼이 이 값인 행만 보기”가 적용됩니다.
- 여러 컬럼에 동시에 필터를 걸 수 있으며, **모든 조건을 만족하는 행만** 남습니다.
- **필터 전체 초기화**는 B2B 전체 강의 화면에서만 보이며, 걸어둔 필터를 모두 해제합니다.

### 4.4 리스트 저장 (저장 버튼)

- 테이블에서 강의를 체크한 뒤 **저장** 버튼을 누르면, “리스트 이름 입력” 창이 뜹니다.
- 리스트 이름, 계약 유형(턴키/인당과금/개별결제), 필요 시 금액 매핑·공유 대상까지 입력 후 저장하면,
- 그 내용이 **브라우저 저장소(LocalStorage)**에 저장되고, 사이드바의 **내 강의리스트** 아래에 새 항목으로 나타납니다.

### 4.5 강의코드로 등록하기

- 리스트 이름, 강의코드(여러 개 입력 가능), 계약 유형을 입력합니다.
- “인당과금”이면 금액 매핑을 입력·실행한 뒤 저장할 수 있습니다.
- 저장 시 역시 브라우저 저장소에 리스트가 저장되고, 사이드바에 반영됩니다.

### 4.6 리스트 관리 (이름 수정 / 강의 추가 / 삭제)

- **이름 수정**: 현재 리스트 이름만 바꿉니다.
- **강의 추가**: “강의 추가”를 누르면 다른 리스트나 전체 목록에서 고른 강의를 이 리스트에 추가하는 흐름으로 전환됩니다.
- **리스트 삭제**: 해당 리스트를 저장소에서 지웁니다.

### 4.7 공유

- **공유하기**: 팀/이름/아이디(사용자 목록에서 자동완성)로 공유 대상을 지정해 저장합니다.
- **공유된 사람 확인**: 이 리스트를 공유받은 사람 목록을 볼 수 있습니다.
- 공유 정보도 브라우저 저장소에 함께 저장됩니다.

### 4.8 세금계산서·정산 관련 (업무요청 / 운영업무)

- **세금계산서 발행요청**, **세금계산서 발행**, **강의료 정산파일 제작** 등을 누르면  
  각각에 맞는 **업무 요청/실행용 창(모달)**이 열립니다.
- 실제 발행·파일 제작은 그 창 안에서 입력하고 진행하는 구조입니다.

### 4.9 대시보드

- **내 강의리스트 → 대시보드**: 저장한 리스트들의 요약(리스트별 강의 수 등)을 보여줍니다.
- **운영 대시보드**: 운영용 지표·데이터를 한 화면에 보여줍니다.
- 둘 다 테이블 대신 **대시보드 전용 화면**이 표시됩니다.

## 5. 데이터가 어디서 오는지 (JSON 파일과 구조)

화면에 보이는 데이터는 아래 JSON 파일들과 브라우저 저장소에서 옵니다.

### 5.1 메인 강의 목록

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/b2b_lectureList.json` | B2B 강의 리스트 전체 (테이블의 기본 데이터) | **배열**. 한 항목이 한 강의. 예시 필드: `카테고리`, `강의명`, `강의코드`, `강의상태`, `B2C강의코드`, `강사명`, `업체명`, `강의생성일`, `정산여부`, `ISBN` 등. |

- “전체 강의”는 여기서 **강의상태가 폐강이 아니고, 강의코드에 ‘daou’가 포함되지 않은** 항목만 사용합니다.
- “제외된 강의”는 폐강이거나 ‘daou’가 포함된 항목만 사용합니다.

### 5.2 사용자 목록 (공유 시 자동완성)

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/users.json` | 공유 대상 선택 시 자동완성용 사용자 목록 | **배열**. 한 항목 예: `{ "team": "팀명", "name": "이름", "id": "아이디" }`. |

### 5.3 교재 정보 (ISBN 매칭)

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/book_list.json` | 강의의 ISBN과 매칭해 교재 정보 표시 | **배열**. 한 항목 예: `seq`, `sapMaterialCode`, `bookTitle`, `author`, `listPrice`, `purchasePrice`, `isbn`. |

### 5.4 Re챔프 강의 정보 (비교·연결)

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/rechamp_lectureList_new.json` | Re챔프 쪽 강의 정보. B2C 강의코드와 매칭 | **배열**. 한 항목 예: `lectureCode`, `category1`~`category4`, `lectureTitle`, `skinId`, `settlementType` 등. |

### 5.5 강의별 정산 상태

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/lecture_settlement_status.json` | 강의코드별 정산 상태 (확인불가/미정산/정산 등) | **객체**. 키가 강의코드(문자열), 값이 상태 문자열. 예: `{ "27684": "확인불가", "27685": "미정산" }`. |

### 5.6 챔프 강의·관리 강의·통합 LCMS (비교·대시보드)

| 파일 경로 | 용도 | 데이터 형태 |
|-----------|------|-------------|
| `data/champ_AdminLecture.json` | 챔프 관리 강의 정보. skinId 등으로 연결 | **배열**. 한 항목 예: `champLectureId`, `adminLectureCode`, `champLectureTitle`, `adminLectureTitle` 등. |
| `data/champ_lecture.json` | 챔프 강의 상세. rowNo 등으로 연결 | **배열**. 한 항목 예: `rowNo`, `lectureType`, `categoryMain`, `categorySub`, `lectureTitle`, `lectureCount` 등. |
| `data/unifiedLcms.json` | 통합 LCMS 강의 정보. 강의 ID로 연결 | **배열**. 한 항목 예: `lectureId`, `categoryLevel1`~`4`, `lectureTitle`, `lectureCount`, `instructors`, `price` 등. |

- 테이블에서 “비교”를 누르거나 행을 펼칠 때, B2B 강의 ↔ Re챔프 ↔ 챔프(AdminLecture·champ_lecture) ↔ 통합 LCMS 순으로 연결해 정보를 보여줍니다.

### 5.7 내 강의리스트·공유 목록 (브라우저 저장소)

- **저장 위치**: 브라우저의 LocalStorage (서버가 아닌 사용자 기기 안).
- **내 강의리스트 한 항목 예시**  
  - `name`: 리스트 이름  
  - `lectureCodes`: 강의코드 배열  
  - `contractType`: 계약 유형 (턴키/인당과금/개별결제)  
  - `priceMap`: 인당과금일 때 강의코드별 금액  
  - `createdAt`, `updatedAt`: 생성·수정 시각  
- **공유 정보**는 별도 키에 “어떤 리스트를 누구에게 공유했는지” 형태로 저장됩니다.
