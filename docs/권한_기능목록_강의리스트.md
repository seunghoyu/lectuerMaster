# 강의리스트 기능 목록 & 권한 매트릭스

이 문서는 현재 `LectureMaster`(강의리스트)에서 제공하는 기능을 **화면/트리거 단위로 상세 나열**하고, 향후 **권한관리(RBAC)** 및 **설정 정책** 설계를 위해 각 기능별 권한 부여안을 함께 정리한다.

## 역할(권한) 정의(초안)

- **운영자(Operator)**  
  일상 운영 업무 수행 중심. 데이터 조회/내 리스트 작업(저장/공유/정산 보조 등)은 가능하되, 시스템에 영향이 큰 작업(삭제/전사 정책 변경)은 제한.

- **관리자(Admin)**  
  팀/조직 운영을 위한 관리 권한. 운영자의 대부분 기능 + 리스트/데이터 관리 작업을 수행 가능.

- **최고관리자(SuperAdmin)**  
  시스템 전체 책임자. 모든 기능 + 향후 “권한/설정/정책” 관리 기능을 포함해 전권.

> 표기 규칙: 아래 표의 **권한 권장**은 “최소 권한 원칙” 기준 초안이며, 실제 조직 정책에 맞게 조정 가능.

## 권한 부여안(추천 정책 초안)

### 운영자(Operator)
- **허용(권장)**: 조회(전체 강의/내 리스트/공유 리스트), 검색/필터/페이지, 리스트 저장/이름수정/공유, 비교 모달
- **제한(권장)**: 리스트 삭제, 정산/세무/파일 생성 계열(운영업무/업무요청)

### 관리자(Admin)
- 운영자 권한 + **리스트 삭제**, **정산/일괄생성 기능**(추후 구현 시) 일부 허용
- “제외된 강의 보기”는 관리자부터 허용하는 구성이 자연스러움

### 최고관리자(SuperAdmin)
- 전 기능 허용 + 추후 추가될
  - **권한관리(역할/사용자 매핑)**
  - **설정(공유 범위/다운로드 정책/감사로그/승인 플로우)**
  - **자동화 실행(세금계산서/정산파일)**

## 권한관리 구현 시 체크리스트(추천)

1. **파괴적 작업 분리**: 삭제/대량 변경/외부 전송(파일/세금계산서)을 별도 권한으로 세분화 가능
2. **공유 정책**: 공유 대상(팀 제한, 외부 공유 차단), 공유 승인/만료, 감사 로그 필요 여부
3. **정산/세무 작업**: 실행자/승인자 분리(2인 승인), 실행 이력 저장(누가/언제/무엇을)
4. **UI 노출 제어**: 권한 없으면 메뉴 자체를 숨기고(또는 disabled), 서버/로컬 검증도 함께 적용
