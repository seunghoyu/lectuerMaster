# 1. 개요 및 범위

## 시스템 범위

- **포함:** B2B 강의리스트 조회·필터·검색, 내 강의리스트 저장/수정/삭제/공유, 강의코드로 등록하기·미정산 강의 등록하기, 리스트별 공유하기·공유된 사람 확인, 세금계산서 발행요청·세금계산서 발행·강의료 정산파일 제작(업무요청·운영업무), 대시보드(내 강의리스트·운영 대시보드), 비교 모달(강의 상세 연계).
- **제외:** 서버 API·백엔드 DB 직접 연동(현재는 JSON 파일·브라우저 저장소 기준). 로그인·역할별 권한 상세는 *미정*.

## 사용자 역할 및 권한 개요

현재 단일 사용자(브라우저 저장소 기준). 역할: **운영자**(리스트 관리·공유·업무요청·운영업무). 관리자/역할별 권한 상세는 *미정*.

## 용어 정의

- **ERD:** 엔티티 관계 다이어그램. 테이블(데이터 집합)과 관계를 도식으로 나타낸 것.
- **엔티티:** 논리적 데이터 단위(테이블 또는 JSON/저장소 키 단위).
- **기능 ID:** 기능 식별자(예: F-001, F-002). 문서 안에서 서로 참조할 때 사용.
- **데이터 원천:** 해당 데이터를 만들거나 갱신하는 주체(파일 경로 또는 앱 내 로직).
