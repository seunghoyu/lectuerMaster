# 내 강의리스트 대시보드 기능 기획안

## 📋 개요

"내 강의리스트" 드롭다운을 클릭했을 때, 개별 리스트를 하나씩 확인하는 방식 대신, **모든 내 강의리스트를 한눈에 볼 수 있는 대시보드 형식의 페이지**를 제공합니다.

---

## 🎯 목표

1. **한눈에 파악**: 모든 내 강의리스트를 블록(카드) 형식으로 한 화면에 표시
2. **빠른 접근**: 각 리스트의 주요 정보를 바로 확인하고 클릭으로 바로 이동
3. **직관적인 UI**: 대시보드 느낌의 깔끔한 레이아웃

---

### **카드 블록 구성**

각 리스트는 **카드(블록) 형식**으로 표시되며, 다음 정보를 포함:

1. **리스트 이름** (큰 글씨, 볼드)
2. **강의 개수** (예: "15개 강의")
3. **생성일** (예: "2024-01-15 10:30")
4. **마지막 수정일** (예: "2024-01-20 14:20")
5. **카테고리 분포** (예: "일반영어: 5개, 일본어: 3개, 중국어: 7개")
6. **공유 상태** (공유된 사람 수 표시, 예: "3명과 공유 중")
7. **빠른 액션 버튼** (선택사항)
   - 리스트 열기
   - 공유하기
   - 삭제하기

### **반응형 그리드**

- **데스크톱**: 한 줄에 3-4개 카드
- **태블릿**: 한 줄에 2개 카드
- **모바일**: 한 줄에 1개 카드

## 🔧 기술적 구현 방안

### **1. 새로운 컴포넌트 생성**

```
javascript/components/
  └── dashboardRenderer.js  # 대시보드 렌더링 컴포넌트
```

**역할:**
- 모든 내 강의리스트를 가져와서 카드 형식으로 렌더링
- 각 리스트의 통계 정보 계산 (강의 개수, 카테고리 분포 등)
- 카드 클릭 시 해당 리스트 상세 페이지로 이동

### **2. CSS 스타일 추가**

```
css/components/
  └── dashboard.css  # 대시보드 스타일
```

**스타일 요소:**
- 그리드 레이아웃 (CSS Grid 또는 Flexbox)
- 카드 디자인 (그림자, 호버 효과)
- 반응형 디자인

### **3. app.js 수정**

**추가할 메서드:**
- `showDashboard()`: 대시보드 화면 표시
- `renderDashboard()`: 대시보드 렌더링
- `setupDashboardClick()`: 사이드바 "내 강의리스트" 클릭 이벤트 처리

**수정할 부분:**
- 사이드바에서 "내 강의리스트" 드롭다운 토글 클릭 시:
  - 기존: 드롭다운만 열림
  - 변경: 드롭다운 열림 + 대시보드 화면 표시

### **4. 데이터 처리**

**StorageService 활용:**
- `StorageService.loadAllLists()`: 모든 리스트 가져오기
- `StorageService.getSharedWithList()`: 각 리스트의 공유 정보 가져오기

**추가 계산:**
- 각 리스트의 카테고리별 강의 개수 계산
- 공유된 사람 수 계산

## 🎨 UI 디자인 가이드

### **카드 스타일**
- 배경: 흰색
- 테두리: 회색 (1px)
- 그림자: 약간의 그림자 효과
- 호버: 마우스 올리면 약간 상승 효과 (transform: translateY(-2px))
- 패딩: 적절한 여백

### **색상**
- 리스트 이름: 진한 회색 (볼드)
- 강의 개수: 중간 회색
- 날짜: 연한 회색
- 카테고리: 각 카테고리별 색상 구분 (선택사항)

### **아이콘**
- 리스트 아이콘: `fa-file-lines`
- 공유 아이콘: `fa-share-nodes`
- 삭제 아이콘: `fa-trash` (호버 시에만 표시)

## 🔄 사용자 플로우

1. **사용자가 사이드바에서 "내 강의리스트" 클릭**
   - 드롭다운이 열림
   - 동시에 대시보드 화면이 표시됨

2. **대시보드에서 리스트 카드 확인**
   - 모든 리스트가 카드 형식으로 표시됨
   - 각 리스트의 주요 정보 확인

3. **카드 클릭**
   - 해당 리스트의 상세 페이지로 이동
   - 기존 `loadSavedList()` 메서드 활용

4. **빠른 액션 버튼 클릭** (선택사항)
   - 공유하기: 공유 모달 열기
   - 삭제하기: 삭제 확인 후 삭제

## 💡 추가 고려사항

### **빈 상태 처리**
- 저장된 리스트가 없을 때:
  - 안내 메시지 표시
  - "리스트 만들기" 버튼 제공 (B2B 강의리스트로 이동)

### **정렬 옵션** (선택사항)
- 최신순 / 오래된순
- 이름순
- 강의 개수순

### **검색 기능** (선택사항)
- 리스트 이름으로 검색
- 필터링 기능

### **성능 최적화**
- 리스트가 많을 경우 가상 스크롤 고려
- 또는 페이지네이션 적용
